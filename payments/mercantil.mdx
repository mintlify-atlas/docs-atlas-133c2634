## Descripción

Mercantil Banco es el principal proveedor de pagos locales, soportando tres métodos:

- **TDD (Tarjeta de Débito)**: Pagos con tarjetas de débito
- **TDC (Tarjeta de Crédito)**: Pagos con tarjetas de crédito
- **C2P (Cliente a Punto)**: Pagos móviles persona a comercio

<Info>
  Ubicación del código: `functions/banking/mercantil/api_mercantil.js`
</Info>

## Métodos de Pago

### 1. Tarjeta de Débito (TDD)

#### Endpoint: `api_mercantil_tdd_pay`

<Steps>
  <Step title="Solicitar Autorización">
    Primero se debe obtener autorización usando `api_mercantil_auth`
  </Step>
  <Step title="Procesar Pago">
    Usar el código de autorización de dos factores para completar el pago
  </Step>
</Steps>

#### Solicitar Autorización

**Function:** `api_mercantil_auth`

```javascript
const response = await fetch('YOUR_FUNCTION_URL/api_mercantil_auth', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    data: {
      customer_id: "V12345678",
      card_number: "501878200066287386",
      ipaddress: "127.0.0.1"

  })

```

```json

    "merchantId": "<merchantId>",

  },
  "client_identify": {
    "ipaddress": "127.0.0.1"
  },
  "transaction_authInfo": {
    "trx_type": "solaut",
    "payment_method": "tdd",
```

#### Procesar Pago TDD

```javascript
const response = await fetch('YOUR_FUNCTION_URL/api_mercantil_tdd_pay', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    data: {
      amount: 15.00,

```

<Warning>
  El CVV y el código 2FA son automáticamente encriptados con AES-256 antes de enviarse al banco
</Warning>

**Código de Implementación:**

```javascript

let data = JSON.stringify({
  "merchant_identify": {
    "integratorId": 1,
    "merchantId": merchantId,
    "terminalId": "1"
  },
  "client_identify": {
    "ipaddress": "127.0.0.1"
  },
  "transaction": {
    "trx_type": "compra",
    "payment_method": "tdd",
    "customer_id": customer_id,
    "card_number": card_number,
    "expiration_date": expiration_date,
    "cvv": cvv_encr,
    "twofactor_auth": twofactor_auth_encr,
    "account_type": account_type,
    "invoice_number": invoice_number,
    "currency": "ves",
    "amount": amount
  }
});
```

### 3. Cliente a Punto (C2P)

Pagos móviles desde cualquier banco venezolano hacia una cuenta Mercantil.

#### Endpoint: `api_mercantil_c2p_pay`

<Steps>
  <Step title="Solicitar Código SCP">
    El cliente debe solicitar un código de autorización usando `api_mercantil_c2p_auth`
  </Step>
  <Step title="Procesar Pago C2P">
    Completar el pago con el código 2FA
  </Step>
  <Step title="Verificar Transacción">
    Opcionalmente buscar la transacción con `api_mercantil_c2p_search`
  </Step>
</Steps>

#### Solicitar Código SCP

**Function:** `api_mercantil_c2p_auth`

```javascript
const response = await fetch('YOUR_FUNCTION_URL/api_mercantil_c2p_auth', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    data: {
      destination_id: "V18367443",
      destination_mobile_number: "584241513063",
      ipaddress: "127.0.0.1"
    }
  })
});
```

<Info>
  La cédula y el teléfono de destino son encriptados automáticamente por la función
</Info>

#### Procesar Pago C2P

```javascript
const response = await fetch('YOUR_FUNCTION_URL/api_mercantil_c2p_pay', {
```

## Conciliación TED

### Endpoint: `api_mercantil`

La función principal procesa archivos TED (Transferencia Electrónica de Datos) para conciliación bancaria.

<Steps>
  <Step title="Listar Lotes">
    Obtiene lista de archivos disponibles para descarga
  </Step>
  <Step title="Descargar Archivo">
    Descarga archivos .7z con movimientos bancarios
  </Step>
  <Step title="Procesar Archivo">
    Extrae y procesa archivos .cif encriptados
  </Step>
  <Step title="Registrar en BD">
    Guarda movimientos en PostgreSQL
  </Step>
</Steps>

**Código de Procesamiento (api_mercantil.js:88-214):**

```javascript
// 1. Listar archivos disponibles
axios.post(endpoint + 'listar-lote', {
  "merchantIdentify": {
    "integratorId": 31,
    "merchantId": merchantId_ted,
    "terminalId": "abcde"
  },
  "fromDate": encryption.encryptAES256("2025-10-01", cifr_ted),
  "toDate": encryption.encryptAES256("2025-10-30", cifr_ted)
});

// 2. Descargar y extraer archivo .7z
const base64Trama = response_descarga.data.fileContent;
const decodedData = Buffer.from(base64Trama, 'base64');
await fs.writeFile(archivePath, decodedData);

// 3. Extraer contenido usando 7zip
const seven = extractFull(archivePath, dir_arc, {
  $bin: pathTo7zip
});

// 4. Desencriptar archivo .cif
const cifContent = await fs.readFile(cifPath);
const decodedCifContent = iconv.decode(cifContent, 'ISO-8859-1');
const valorarchivo = encryption.decryptAES256(decodedCifContent, cifr_ted);

// 5. Parsear movimientos
let fecha_reg = valorarchivo.substr(17, 8);
let num_cta = valorarchivo.substr(41, 20);
let saldo_total = valorarchivo.substr(102, 17);
// ... más campos

// 6. Registrar en PostgreSQL
const conc_conciliado_b = await dbpostgres.sqltmt(
  "insert", 
  "conc_conciliado_b", 
  campos_cociliado_b, 
  null, null, null, null, 
  valores_cociliado_b
);
```

## Seguridad

<Warning>
  **Datos Sensibles Encriptados:**
  - CVV de tarjetas
  - Códigos de autenticación 2FA
  - Números de teléfono
  - Cédulas de identidad
  - Contenido de archivos TED
</Warning>

### Ejemplo de Encriptación

```javascript
const encryption = require("../../../config/encryption");
const cifr = config.cifr;

// Encriptar
const cvv_encr = encryption.encryptAES256("752", cifr);
const twofactor_auth_encr = encryption.encryptAES256("11111111", cifr);

// Desencriptar
const decrypted = encryption.decryptAES256(encrypted_data, cifr);
```

## Headers Requeridos

Todas las peticiones a Mercantil requieren:

```javascript
headers: {
  'X-IBM-Client-ID': client_id,  // o client_id / clientidibmpm
  'Content-Type': 'application/json'
}
```

<Tip>
  Para pagos móviles usar `clientidibmpm` en lugar de `client_id`
</Tip>
