## Overview

This endpoint generates tickets for a specific event. It creates tickets for all configured zones with the specified number of seats, storing them in both Firestore and PostgreSQL databases.

## Request

<ParamField body="data" type="object" required>
  Request data object
  
  <ParamField body="data.idevent" type="string" required>
    The unique identifier of the event for which to generate tickets
  </ParamField>
</ParamField>

## Ticket Structure

Each generated ticket includes:

- `ticket_id`: Unique identifier (format: `{event_id}-{generated_id}`)
- `event_id`: Reference to the parent event
- `event_name`: Name of the event
- `seat_id`: Seat identifier (format: `{zone_id}-{seat_number}`)
- `zone`: Zone name
- `color`: Zone color code
- `seat_row`: Initial value "por asignar" (to be assigned)
- `status`: Ticket availability status (initially `true`)
- `date_start`: Event start date
- `date_end`: Event end date
- `ledger`: Activity log with initial "generated" action
- `date.created`: Ticket creation timestamp
- `date.updated`: Last update timestamp

## Notes

- The endpoint requires the event to have configured zones in the `events/{event_id}/setup/zones` collection
- Zones must have `status: true` to allow ticket generation
- Each zone must specify the number of `seats` to generate
- Tickets are created in batches for performance optimization
- The ledger field tracks all ticket lifecycle events with timestamps
- Both Firestore and PostgreSQL databases are updated in the same transaction
