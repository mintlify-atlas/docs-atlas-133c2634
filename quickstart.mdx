## Introducción

Esta guía te ayudará a realizar tu primera llamada a la API de TMT, un sistema de gestión de tickets basado en Firebase Cloud Functions.

<Note>
  La API de TMT utiliza Firebase Authentication para la autenticación de usuarios y Firebase Cloud Functions para el procesamiento de solicitudes.
</Note>

## Prerequisitos

Antes de comenzar, asegúrate de tener:

- Node.js 22 o superior instalado
- Credenciales de Firebase para tu proyecto
- Un cliente HTTP como cURL, Postman, o Axios

## Configuración Inicial

<Steps>
  <Step title="Instala las dependencias necesarias">
    Si estás integrando TMT en tu aplicación, instala los paquetes de Firebase:

    <CodeGroup>
      ```bash npm
      npm install firebase firebase-admin cors
      ```

      ```bash yarn
      yarn add firebase firebase-admin cors
      ```

      ```bash pnpm
      pnpm add firebase firebase-admin cors
      ```
    </CodeGroup>
  </Step>

  <Step title="Configura Firebase en tu proyecto">
    Inicializa Firebase con tus credenciales:

    ```javascript
    const { initializeApp } = require('firebase/app');
    const { getAuth } = require('firebase/auth');

    const firebaseConfig = {
      apiKey: "YOUR_API_KEY",
      authDomain: "YOUR_AUTH_DOMAIN",
      projectId: "YOUR_PROJECT_ID",
      storageBucket: "YOUR_STORAGE_BUCKET",
      messagingSenderId: "YOUR_MESSAGING_SENDER_ID",
      appId: "YOUR_APP_ID"
    };

    const app = initializeApp(firebaseConfig);
    const auth = getAuth(app);
    ```
  </Step>

  <Step title="Autentica al usuario">
    Obtén un token de autenticación de Firebase:

    ```javascript
    const { signInWithEmailAndPassword } = require('firebase/auth');

    async function authenticate(email, password) {
      try {
        const userCredential = await signInWithEmailAndPassword(auth, email, password);
        const idToken = await userCredential.user.getIdToken();
        return idToken;
      } catch (error) {
        console.error('Error de autenticación:', error);
        throw error;
      }
    }
    ```
  </Step>

  <Step title="Realiza tu primera llamada a la API">
    Ahora puedes llamar a las Cloud Functions de TMT:

    ```javascript
    const axios = require('axios');

    async function callTMTAPI(endpoint, data, idToken) {
      const response = await axios.post(
        `https://<region>-<project-id>.cloudfunctions.net/${endpoint}`,
        { data },
        {
          headers: {
            'Content-Type': 'application/json',
            'Authorization': `Bearer ${idToken}`
          }
        }
      );
      return response.data;
    }
    ```
  </Step>
</Steps>

### Llamada a la API

<CodeGroup>
  ```javascript Node.js
  const axios = require('axios');

  async function createClient() {
    const endpoint = 'create_client';
    const baseURL = 'https://<region>-<project-id>.cloudfunctions.net';
    
    try {
      const response = await axios.post(
        `${baseURL}/${endpoint}`,
        clientData,
        {
          headers: {
            'Content-Type': 'application/json'
          }
        }
      );
      
      console.log('Cliente creado exitosamente:', response.data);
      return response.data;
    } catch (error) {
      console.error('Error al crear cliente:', error.response?.data || error.message);

    }
  }

  createClient();
  ```

  ```python Python
      endpoint = 'create_client'

      client_data = {
          "data": {
              "name": "Empresa Ejemplo C.A.",
              "email": "contacto@empresaejemplo.com",
              "password": "SecurePassword123!",
              "name_commercial": "Empresa Ejemplo",
              "phone": "+58-412-1234567",
              "status": True,
              "id": "J-12345678-9",
              "id_type": "RIF",
              "city": "Caracas",
              "country": "Venezuela",
  ```

### Respuesta Exitosa

Cuando el cliente se crea exitosamente, recibirás una respuesta como esta:

```json
{
  "message": "Ingresado con el ID: abc123xyz456def789.",
  "status": 200,
  "data": {
    "uid": "abc123xyz456def789"
  }
}
```

<Info>
  El `uid` devuelto es el identificador único del usuario en Firebase Authentication y se utiliza como clave del documento en la colección `u_clients` de Firestore.
</Info>

### Respuesta de Error

Si hay un error al crear el cliente, recibirás:

```json
{
  "message": "Error creando nuevo usuario:",
  "status": 400,
  "data": {
    "error": {
      "code": "auth/email-already-exists",
      "message": "The email address is already in use by another account."
    }
  }
}
```

## Datos Almacenados

Según el código en `functions/users/create_client.js:16-49`, cuando creas un cliente, se almacenan los siguientes datos en Firestore:

- **Información básica**: nombre, email, teléfono, estado
- **Dirección**: ciudad, país, línea de dirección, código postal
- **Información fiscal**: tipo de cuenta, registro comercial, licencia comercial, tipo de contribuyente
- **Representante legal y persona de contacto**
- **Cuentas bancarias**
- **Timestamps**: fecha de creación, última actualización, último acceso
- **Contadores**: colaboradores, contratos, eventos (inicializados en 0)
