## User Types

TMT supports three distinct user types, each with specific data structures and stored in separate Firestore collections:

## Key Differences Between User Types

<CardGroup cols={3}>
  <Card title="Clients">
    - Most comprehensive data structure
    - Business and fiscal information
    - Legal representative details
    - Bank account information
    - Usage statistics tracking
  </Card>
  
  <Card title="Collaborators">
    - Linked to a parent client
    - Simplified data structure
    - No business information
    - Personal identification only
  </Card>
  
  <Card title="Staff">
    - Independent of clients
    - 2FA security settings
    - Timezone configuration

    - Platform administration access
  </Card>

## Creating Users

All user creation follows the same pattern:

<Steps>
  <Step title="Create Firebase Auth User">
    First, create the authentication user with email and password.
    
    ```javascript
    const userRecord = await auth.createUser({
      email: email,
      password: password
    });
    const uid = userRecord.uid;
    ```
  </Step>
  
  <Step title="Store User Data in Firestore">
    Use the UID as the document ID in the appropriate collection.
    
    ```javascript
    await db.collection('u_clients').doc(uid).set(userData);
    ```
  </Step>
  
  <Step title="Return UID to Client">
    Respond with the newly created UID.
    
    ```javascript
    response.json({ 
      message: `Ingresado con el ID: ${uid}`,
      status: 200,
      data: { uid: uid }
    });
    ```
  </Step>
</Steps>

## Common Patterns

### Address Structure

All user types share the same address structure:

```javascript
address: {
  city: string,
  country: string,
  line: string,
  zipcode: string
}
```

### Date Tracking

All user types track creation and update timestamps:

```javascript
date: {
  create: Timestamp,      // Set on creation
  last_update: string,    // Empty initially
  last_access: string     // Empty initially
}
```

### Error Handling

All creation functions follow the same error pattern:

<CodeGroup>
```javascript Error Handling
auth.createUser({ email, password })
  .then((userRecord) => {
    uid = userRecord.uid;
    // Continue with Firestore write
  })
  .catch((error) => {
    console.log('Error creating new user:', error);
    response.json({
      message: 'Error creando nuevo usuario:',
      status: 400,
      data: { error: error }
    });
  });
```
</CodeGroup>

## Best Practices

<CardGroup cols={2}>
  <Card title="UID as Document ID" icon="key">
    Always use Firebase Auth UID as the Firestore document ID for direct user-to-data linking.
  </Card>
  
  <Card title="Timestamp Creation" icon="clock">
    Use `Timestamp.now()` for the creation date to ensure server-side timestamp accuracy.
  </Card>
  
  <Card title="Validate Before Creation" icon="shield-check">
    Check if email already exists before creating a new user to prevent duplicates.
  </Card>
  
  <Card title="Secure Passwords" icon="lock">
    Enforce strong password requirements on the client side before submission.
  </Card>
</CardGroup>

<Tip>
  When creating collaborators, always validate that the parent client UID exists before proceeding with user creation.
</Tip>
